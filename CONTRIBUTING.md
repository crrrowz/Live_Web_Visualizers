## 🚀 كيف تساهم؟

### الإبلاغ عن مشاكل (Issues)

1. تأكد أن المشكلة لم يتم الإبلاغ عنها سابقاً
2. افتح Issue جديد مع:
   - **عنوان واضح** يصف المشكلة
   - **خطوات إعادة الإنتاج** — كيف نُعيد المشكلة
   - **السلوك المتوقع** vs **السلوك الفعلي**
   - **بيئة التشغيل** — المتصفح، نظام التشغيل، Node.js
   - **لقطة شاشة** إن أمكن

### اقتراح ميزات

افتح Issue بعنوان يبدأ بـ `[اقتراح]` واشرح:
- ما الميزة المقترحة؟
- لماذا هي مفيدة؟
- كيف تتصور تنفيذها؟

### إرسال Pull Request

1. **Fork** المستودع
2. استنسخ نسختك:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Live_Web_Visualizers.git
   cd Live_Web_Visualizers
   npm install
   ```
3. أنشئ فرعاً جديداً:
   ```bash
   git checkout -b feature/اسم-الميزة
   ```
4. نفّذ تعديلاتك واختبرها محلياً
5. أنشئ Commit:
   ```bash
   git commit -m "إضافة: وصف التعديل"
   ```
6. ارفع الفرع:
   ```bash
   git push origin feature/اسم-الميزة
   ```
7. افتح **Pull Request** مع وصف واضح

## 🔧 إعداد بيئة التطوير

```bash
# استنساخ المشروع
git clone https://github.com/YOUR_USERNAME/Live_Web_Visualizers.git
cd Live_Web_Visualizers

# تثبيت المكتبات
npm install

# تشغيل خادم التطوير
npm run dev
```

الخادم يعمل على `http://localhost:3000`

### هيكل الملفات المهمة

| الملف | الوظيفة |
|-------|---------|
| `server.js` | خادم Express + REST API |
| `public/js/app.js` | المحرك الرئيسي |
| `public/js/config.js` | **جميع الإعدادات** — عدّل هنا أولاً |

| `public/js/study.js` | نظام التتبع الدراسي |
| `public/css/style.css` | أنماط التصميم |

## 📝 اصطلاحات الكود

### عام
- استخدم **ES Modules** (`import`/`export`)
- استخدم `const` و `let` بدل `var`
- أضف تعليقات بالعربية أو الإنجليزية
- حافظ على تنسيق الكود المتسق (4 مسافات)

### Three.js
- نظّف الموارد دائماً في `dispose()` (geometry, material, texture)
- استخدم `CONFIG` لجميع القيم القابلة للضبط
- اتبع نمط المشاهد الموجودة عند إنشاء مشهد جديد

### CSS
- استخدم متغيرات CSS للألوان المتكررة
- حافظ على دعم RTL
- استخدم `backdrop-filter` و `glassmorphism` للعناصر الشفافة

## 🎨 إنشاء مشهد جديد

لإضافة مشهد جديد، أنشئ ملفاً في `public/js/scenes/` يصدّر هذه الدوال:

```javascript
import * as THREE from 'https://esm.sh/three@0.136.0';

let myObjects = [];

// إنشاء عناصر المشهد
export function create(scene) {
    const geometry = new THREE.SphereGeometry(1, 32, 32);
    const material = new THREE.MeshBasicMaterial({ color: 0x00f0ff });
    const mesh = new THREE.Mesh(geometry, material);
    scene.add(mesh);
    myObjects.push(mesh);
}

// تحديث كل إطار — يُستدعى ~60 مرة/ثانية
export function update(time, bass, rawBass, smoothBass, camera) {
    // حرّك الأجسام بناءً على الصوت
    return {
        bg: new THREE.Color(0x07070d),  // لون الخلفية
        collision: false,                // هل يوجد تصادم؟
        managesCamera: false             // هل المشهد يتحكم بالكاميرا؟
    };
}

// تحديث عند تغيّر المستوى
export function updateLevel(level) { }

// تأثير التصادم (اختياري)
export function onCollision(bass) { }

// تنظيف عند التبديل — مهم جداً!
export function dispose(scene) {
    myObjects.forEach(obj => {
        scene.remove(obj);
        if (obj.geometry) obj.geometry.dispose();
        if (obj.material) obj.material.dispose();
    });
    myObjects = [];
}
```

ثم سجّله في مصفوفة `SCENES` في `app.js`.

## 💬 اصطلاحات Commit

| البادئة | الاستخدام | مثال |
|---------|----------|------|
| `إضافة:` | ميزة جديدة | `إضافة: مشهد البركان` |
| `إصلاح:` | إصلاح خطأ | `إصلاح: تسرب ذاكرة في المجرة` |
| `تحسين:` | تحسين أداء | `تحسين: تقليل عدد draw calls` |
| `توثيق:` | تحديث وثائق | `توثيق: إضافة شرح API` |
| `تنظيف:` | إعادة هيكلة | `تنظيف: فصل الشيدرات لملفات` |
| `اختبار:` | إضافة اختبارات | `اختبار: اختبارات وحدة للـ API` |

## 🔍 مراجعة الكود

عند مراجعة Pull Request نتأكد من:

- ✅ الكود يعمل بدون أخطاء
- ✅ لا يكسر الميزات الموجودة
- ✅ يتبع اصطلاحات الكود
- ✅ المشاهد الجديدة تنظّف مواردها في `dispose()`
- ✅ الإعدادات الجديدة مضافة في `config.js`
- ✅ واجهة المستخدم تدعم RTL
- ✅ التعليقات واضحة ومفيدة
